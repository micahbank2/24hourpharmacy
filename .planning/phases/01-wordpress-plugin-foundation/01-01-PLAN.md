**class-settings.php** — Implement `Pharmacy_Tools_Settings` class with:

1. Class constants: `OPTION_GROUP = 'twentyfourhour_settings'`, `PAGE_SLUG = 'twentyfourhour-settings'`.

2. `register()` method: hooks `add_admin_page` to `admin_menu`, hooks `register_settings` to `admin_init`.

3. `add_admin_page()`: calls `add_options_page()` with title "24Hr Pharmacy Tools", capability `manage_options`, slug `self::PAGE_SLUG`, callback `render_page`.

4. `register_settings()`: Register these settings sections and fields using WordPress Settings API:
   - **Section: API Keys** (`twentyfourhour_api_section`)
     - `twentyfourhour_google_maps_key` — Google Maps API Key (text field)
   - **Section: Savings Card** (`twentyfourhour_savings_section`)
     - `twentyfourhour_lowermyrx_bin` — LowerMyRx BIN
     - `twentyfourhour_lowermyrx_pcn` — LowerMyRx PCN
     - `twentyfourhour_lowermyrx_group` — LowerMyRx Group
   - **Section: Affiliate IDs** (`twentyfourhour_affiliate_section`)
     - `twentyfourhour_amazon_pharmacy_affiliate_id` — Amazon Pharmacy Affiliate ID
     - `twentyfourhour_singlecare_affiliate_id` — SingleCare Affiliate ID

   Each field: `register_setting()` with `'sanitize_callback' => 'sanitize_text_field'`, then `add_settings_field()` with a reusable `render_text_field` callback that reads `get_option()` and outputs `<input type="text" name="..." value="..." class="regular-text" />` with `esc_attr()` escaping.

5. `render_page()`: Check `current_user_can( 'manage_options' )`. Output `<div class="wrap">`, `<h1>`, `<form method="post" action="options.php">`, call `settings_fields()`, `do_settings_sections()`, `submit_button()`.

6. Private `render_text_field( $args )` callback: reads option key from `$args['option']`, outputs escaped input field.

Keep ABSPATH guard. Docblock every method.

**class-shortcodes.php** — Implement `Pharmacy_Tools_Shortcodes` class with:

1. `register()` method: hooks `register_scripts` to `wp_enqueue_scripts`, then registers shortcodes:
   - `add_shortcode( 'medical_disclaimer', ... )`
   - `add_shortcode( 'affiliate_disclosure', ... )`
   - `add_shortcode( 'pharmacy_finder', ... )`
   - `add_shortcode( 'discount_card', ... )`

2. `register_scripts()`: Use `wp_register_script()` (NOT enqueue) for:
   - Handle `'pharmacy-finder'`, path `PHARMACY_TOOLS_URL . 'assets/js/pharmacy-finder.js'`, no deps, version via `filemtime()` with fallback to `PHARMACY_TOOLS_VERSION`, in footer (`true`).
   - Handle `'discount-card'`, path `PHARMACY_TOOLS_URL . 'assets/js/discount-card.js'`, same pattern.
   - Only register if the JS file exists (`file_exists()` check). This prevents errors before Phase 3 builds the widgets.

3. `medical_disclaimer( $atts )`: Process `$atts` with `shortcode_atts( array(), $atts, 'medical_disclaimer' )`. RETURN (not echo) a `<div class="pharmacy-disclaimer" role="note">` containing the disclaimer text from research. Use `esc_html__()` with text domain.

4. `affiliate_disclosure( $atts )`: Same pattern. RETURN `<div class="pharmacy-disclosure" role="note">` with FTC disclosure text.

5. `pharmacy_finder_shortcode( $atts )`: Call `wp_enqueue_script( 'pharmacy-finder' )` to trigger conditional load. Process atts with default `'height' => '400px'`. RETURN `<div id="pharmacy-finder-root" class="pharmacy-widget pharmacy-finder-widget" style="min-height:{height}">` with `<noscript>` fallback. Use `esc_attr()` on height.

6. `discount_card_shortcode( $atts )`: Same pattern with handle `'discount-card'`, root ID `discount-card-root`, default height `300px`.

Keep ABSPATH guard. Docblock every method.

**class-schema.php** — Implement `Pharmacy_Tools_Schema` class with:

1. `register()` method: hooks `output_schema` to `wp_head`.

2. `output_schema()`: Add `if ( is_admin() ) { return; }` guard at top. Then:
   - If `is_singular( 'pharmacy' )` → call `output_pharmacy_schema()`
   - If `is_singular( 'city' )` → call `output_city_schema()`
   - If `is_front_page()` → call `output_website_schema()`
   - Always call `output_webpage_schema()` for any singular page.
   - Always call `output_faqpage_schema()` for any singular page (stub for Phase 7 population).

3. `output_pharmacy_schema()`: Build array with `@context => https://schema.org`, `@type => array( 'Pharmacy', 'LocalBusiness' )`, `name => get_the_title()`, `url => get_permalink()`. Output via `printf( '<script type="application/ld+json">%s</script>', wp_json_encode( $schema, JSON_UNESCAPED_UNICODE | JSON_UNESCAPED_SLASHES ) )`.

4. `output_city_schema()`: `@type => WebPage`, `name`, `url`, `description => get_the_excerpt()`.

5. `output_website_schema()`: `@type => WebSite`, `name => get_bloginfo('name')`, `url => home_url('/')`.

6. `output_webpage_schema()`: Only fires on `is_singular()`. `@type => WebPage`, `name => get_the_title()`, `url => get_permalink()`. Skip if pharmacy or city (already have their own more specific schema).

7. `output_faqpage_schema()` — **Phase 1 stub, required by INFRA-04/ROADMAP.md deliverables:** Only fires on `is_singular()`. Build array with `@type => FAQPage`, `mainEntity => array()` (empty — populated in Phase 7). Output via `wp_json_encode()`. This stub registers the schema type now so Phase 7 can populate it without modifying the hook structure.

All schema output uses `wp_json_encode()`, never manual string concatenation. Keep ABSPATH guard. Docblock every method.
